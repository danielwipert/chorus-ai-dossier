01 — What it is

What is Chorus AI?

Chorus AI is not a chatbot, agent, or general-purpose LLM wrapper. It is a deterministic document intelligence pipeline with strict contracts between stages — designed to produce analytical reports that are grounded, verifiable, and auditable.

The problem

LLMs hallucinate. Single-model outputs are unverified.

A single frontier model summarising a complex document will confidently assert things not in the source. There is no internal check, no cross-validation, no audit trail.

The solution

Three summaries, one fact list, a closed verification loop.

Chorus extracts atomic facts first, then generates summaries independently across multiple model families. Each summary is verified against the locked fact list — any contradiction fails the summary and triggers a regeneration.

The output

A structured dossier with a full audit trail.

The final dossier contains an executive overview, key claims with fact IDs, compiled synthesis, external context, risks, and a complete audit trail — every claim traces to a source span.

Design philosophy

Think CIA intelligence brief meets New Yorker book review.

Grounded facts. Traceable claims. Explicitly labeled uncertainty. No speculation passed off as analysis. If confidence cannot be achieved, the pipeline halts visibly — not silently.

Hard guarantee: The FactList is locked after extraction. No downstream stage can modify it. Summaries and analysis must conform to the facts — not the other way around.

02 — Design theory

Seven non-negotiable design rules.

These govern every implementation decision. They are constraints, not guidelines — a stage that violates them is wrong by definition.

Rule 01
Determinism

Same input + same config = same output. Temperature is zero across all models. No random seeds, no non-deterministic branches.

Rule 02
Grounding First

No summary, analysis, or narrative exists without being grounded in extracted facts. External context is allowed exactly one designated section.

Rule 03
Fail-Closed

Any broken stage gate halts the entire run. There is no silent degradation. A partial dossier is not a dossier.

Rule 04
Traceability

Every output traces back to source spans in the original document. Every key claim references a Fact ID.

Rule 05
Separation of Concerns

Facts, summaries, external context, and opinion are never mixed. Each lives in an explicitly typed artifact that cannot be mutated after creation.

Rule 06
Auditability

Every decision is inspectable after the fact. The audit trail records model participation, contradiction scores, coverage scores, and which summaries contributed to each section.

Rule 07
No Agent Autonomy

No background retries beyond configured limits, no self-directing behavior, no tool-calling loops. V1 is a pipeline, not an agent.

04 — The verification core

The closed-loop verification core.

Stage 4 is the hallucination gate. It is the reason three summaries are generated rather than one. A summary that contradicts any extracted fact fails — regardless of how plausible it sounds.

Closed-Loop Verification Flow
FactList
locked after Stage 2
→
Summary A/B/C

→
Verifier
Model 5
→
Scores each fact
aligned / absent / contradicted
→
Pass / Retry
per summary
↺ retry loop — failed summaries only, FactList never changes
PASS

≥ 2 summaries have contradiction_score ≤ threshold (default 0.0). Run proceeds to Stage 5.

RETRY

Fewer than 2 summaries pass. Failed summaries are regenerated and re-verified. FactList is unchanged.

HALT

Max retries reached without 2 passing. Run halts. Diagnostic artifact written. All upstream outputs preserved.

Primary metric: contradiction_score = contradicted_facts / total_facts. Coverage (how many facts the summary mentions) is recorded as a secondary metric for the audit trail — but a tight summary that skips facts without contradicting them is acceptable. Only contradictions are disqualifying.

Architecture and artifact contracts.

Every stage passes typed JSON artifacts. No free-form text between stages. Artifacts are written to disk immediately upon creation and are never mutated after that.
